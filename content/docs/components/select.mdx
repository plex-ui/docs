### Custom views

<ComponentPreview details={<>You can fully customize the styling and rendering of select's trigger and options:<br/><ul><li><strong>Trigger</strong> &ndash; <code>triggerClassName</code>, <code>TriggerStartIcon</code>, and <code>TriggerView</code></li><li><strong>Options</strong> &ndash; <code>optionClassName</code>, <code>OptionView</code>, as well as <code>description</code> and <code>tooltip</code> fields in <code>options</code></li></ul></>} code={`const RoleOptionDescription = ({ children }: { children: ReactNode }) => (
  <div className="font-normal text-secondary text-[0.935em] leading-[1.45]">
    {children}
  </div>
);

const roles = [
  {
    value: "owner",
    label: "Owner",
    description: (
      <RoleOptionDescription>
        Can modify project information and manage project members
      </RoleOptionDescription>
    ),
  },
  {
    value: "reader",
    label: "Reader",
    description: (
      <RoleOptionDescription>
        Can make API requests that read or modify data
      </RoleOptionDescription>
    ),
  },
];

<Select
  value={role}
  options={roles}
  placeholder="Select role..."
  align="start"
  listMinWidth={260}
  variant="ghost"
  size="lg"
  onChange={({ value }) => setRole(value)}
  TriggerStartIcon={role === "owner" ? UserLock : User}
  triggerClassName="font-semibold"
  optionClassName="font-semibold"
/>`} previewMinHeight={168}>
  <SelectCustomViewsDemo />
</ComponentPreview>

### Actions

### Grouped options

<ComponentPreview details={<>Select supports option groups, similar to the native <code>&lt;optgroup&gt;</code> element. All selects are searchable by default, but will only present a searchbox when it makes sense. Currently, this means when there are 15 or more options.</>} code={`<Select
  value={value}
  options={groupedItems}
  onChange={(v) => setValue(v.value)}
  variant="outline"
  size="lg"
  side="bottom"
  listMinWidth={300}
  searchPlaceholder="Select an option..."
  clearable
/>`} previewMinHeight={168}>
  <SelectGroupedOptionsDemo />
</ComponentPreview>

### Multiple selection

<ComponentPreview details={<>Multiple selection is supported through the <code>multiple</code> property. This changes how <code>value</code>, <code>onChange</code>, and <code>TriggerView</code> are typed, accepting and returning arrays of values.</>} code={`<Select
  variant="soft"
  placeholder="Select fruits..."
  options={fruitsOptions}
  name="fruits"
  multiple
  clearable
  value={fruits}
  onChange={(values) => setFruits(values.map(({ value }) => value))}
  TriggerView={MultiFruitTriggerView}
/>`} previewMinHeight={168}>

</ComponentPreview>
