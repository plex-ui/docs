### Open on hover

<ComponentPreview details={<>Use <code>showOnHover</code> to create popovers that appear on hover and focus events. This should be used sparingly and with clear intention.<br/><br/>By default, hover popovers will not alter the focused element on the page, nor restore focus to the trigger on close. However, if interactive content is present in the popover, and <code>tab</code> is used to navigate into the popover, focus will be restored to the trigger on close.</>} code={`<Popover showOnHover>

### Nesting

All floating UI components in Plex UI can be infinitely nested by relying on `z-index: auto;` and natural stacking order.

Additionally, `esc` keypresses are handled independently as a stacking order.

## Reference

**Popover** (root)

<PropsTable data={[
  { prop: "showOnHover", type: "boolean", description: "Open on hover and focus instead of click" },
]} />

**Popover.Trigger** — Renders the element that toggles the popover. The trigger should be the user's point of interaction for revealing the popup. No specific props.

**Popover.Content**

<PropsTable data={[
  { prop: "width", type: "number | string", description: "Panel width (px or CSS value)" },
  { prop: "minWidth", type: "number | string", default: "300", description: "Minimum panel width in px" },
  { prop: "maxWidth", type: "number | string", description: "Maximum panel width" },
  { prop: "side", type: '"top" | "right" | "bottom" | "left"', description: "Preferred placement" },
  { prop: "align", type: '"start" | "center" | "end"', description: "Alignment relative to trigger" },
  { prop: "sideOffset", type: "number", description: "Offset from trigger (px)" },
  { prop: "alignOffset", type: "number", description: "Alignment offset (px)" },
]} />

**usePopoverController()** — Hook available inside `Popover.Content`. Returns `{ close: () => void, shake: () => void }`.
