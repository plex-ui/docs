### Sizing & roundness

<ComponentPreview details={<>Sizing of the input is controlled with <code>size</code> and <code>gutterSize</code> props. <code>size</code> sets the input to a specific control <code>height</code>, and <code>gutterSize</code> sets the horizontal padding. Each <code>size</code> provides default values for <code>gutterSize</code>, as well as <code>font-size</code>. Each of these values can be overridden independently.</>} code={`<div className="flex flex-col gap-2.5 justify-start items-start w-[320px]">
  <Input placeholder="Enter text..." size="lg" />
  <Input placeholder="Search..." startAdornment={<Search className="fill-tertiary" />} size="lg" />
  <Input value={username} placeholder="Username" onChange={(evt) => setUsername(evt.target.value)} maxLength={16} endAdornment={<span className="mt-[1.5px] tabular-nums text-tertiary text-[.875em]">{username.length}/16</span>} />
</div>`} previewMinHeight={168}>
  <InputSizingDemo />
</ComponentPreview>

### Start adornment

<ComponentPreview details={<>You can add arbitrary content to the beginning of an input with <code>startAdornment</code> &ndash; like a search icon. Ideally, the spacing will look pretty good by default. If you need to customize the spacing: <code>gutterSize</code> determines the amount of spacing on the edges of the container, which you can fine-tune as needed (if you need precise spacing not supported by <code>gutterSize</code>, use negative left margin on your adornment directly, e.g. <code>-ml-2</code>). The spacing between the adornment and the input is controlled with <code>gap</code> on the container, which you can customize as needed with Tailwind classes.</>} code={`<Input
  placeholder="Start adornment"
  startAdornment={<Search className="fill-tertiary" />}
/>`} previewMinHeight={168}>
  <div className="w-[320px]">
    <Input
      placeholder="Start adornment"
      startAdornment={<Search className="fill-tertiary" />}
    />
  </div>
</ComponentPreview>

### Clear button

<ComponentPreview details={<>Use the <code>onClear</code> prop to display a built-in clear button. The button appears when the input has a value and is not disabled or read-only.</>} code={`<Input

  onChange={(evt) => setValue(evt.target.value)}

  <InputEndAdornmentDemo />

### End adornment

<ComponentPreview details={<>You can add arbitrary content to the end of an input with <code>endAdornment</code> &ndash; like a character counter or custom icon. The same spacing guidance from <code>startAdornment</code> applies here.</>} code={`<Input
  placeholder="Username"
  value={username}
  onChange={(evt) => setUsername(evt.target.value)}
  maxLength={16}
  endAdornment={
    <span className="mt-[1.5px] tabular-nums text-tertiary text-[.875em]">
      {username.length}/16
    </span>
  }
/>`} previewMinHeight={168}>
  <div className="w-[320px]">
    <Input
      placeholder="Username"
      defaultValue=""
      maxLength={16}
      endAdornment={
        <span className="mt-[1.5px] tabular-nums text-tertiary text-[.875em]">0/16</span>
      }
    />
  </div>
</ComponentPreview>

### Invalid

<ComponentPreview details={<>Use the <code>invalid</code> prop to mark the input as invalid. You decide how this state is removed, whether that's through <code>onFocus</code>, <code>onChange</code>, an <code>onSubmit</code> handler, etc. Default visual styles will be applied, based on the <code>variant</code>. Additionally, <code>[data-invalid]</code> is applied to the container for custom styling.</>} code={`<Input invalid placeholder="Invalid input" />`} previewMinHeight={168}>
  <InputInvalidDemo />
</ComponentPreview>

Plex UI also provides a `FieldError` component (`@plexui/ui/components/FieldError`) that pairs well with invalid inputs — it renders an error icon with a message styled for form validation.

### Optical alignment

<ComponentPreview details={<>Use <code>opticallyAlign</code> with either <code>"start"</code> or <code>"end"</code> to offset the gutter, aligning the input text with surrounding content. Works with <code>Field</code> — pass <code>opticallyAlign</code> on the Field and it forwards to the child control automatically.</>} code={`<div className="flex flex-col gap-3 w-[400px]">
  <div className="border border-dashed border-alpha/20 rounded-md py-6 px-8">
    <div className="mb-5 text-secondary text-sm">Default</div>
    <div className="flex flex-col gap-5">
      <Field label="Field 1">
        <Input placeholder="Sample input" pill />
      </Field>
      <Field label="Field 2">
        <Input placeholder="Sample input" pill />
      </Field>
    </div>
    <Button className="mt-6" color="primary" variant="soft" pill>Submit</Button>
  </div>
  <div className="border border-dashed border-alpha/20 rounded-md py-6 px-8">
    <div className="mb-5 text-secondary text-sm">opticallyAlign="start"</div>
    <div className="flex flex-col gap-5">
      <Field label="Field 1" opticallyAlign="start">
        <Input placeholder="Sample input" pill />
      </Field>
      <Field label="Field 2" opticallyAlign="start">
        <Input placeholder="Sample input" pill />
      </Field>
    </div>
    <Button className="mt-6" color="primary" variant="soft" pill>Submit</Button>
  </div>
</div>`} previewMinHeight={168}>
  <div className="flex flex-col gap-3 w-[400px]">

Input shares the unified 9-step size scale with Button, Select, SelectControl, and SegmentedControl — set the same `size` on adjacent controls and they'll align perfectly.
