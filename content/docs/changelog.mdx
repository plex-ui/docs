## 0.17.1 — 2026-05-04

### Fixed

<ul className="changelog-list">
  <li>
    <strong>Text color consistency</strong> on <code>SelectControl</code> and <code>TagInput</code>. Both components were missing a base <code>color</code> rule and inherited the page foreground (~<code>#181818</code>), which sat one step lighter than the <code>--color-text</code> token used by <code>Input</code>, <code>Textarea</code>, and the <code>TagInput</code> tag chips themselves. Side-by-side controls in a form now share the same value text color (<code>--input-text-color</code>). Placeholder state on <code>SelectControl</code> (<code>data-selected="false"</code>) is unchanged.
  </li>
</ul>

## 0.17.0 — 2026-05-02

### Added

<ul className="changelog-list">
  <li>
    <strong>Vite plugin</strong> — new sub-export <code>@plexui/ui/vite</code>. One line in <code>vite.config.ts</code> replaces every install-time gotcha: pins <code>css.transformer = 'postcss'</code>, pins <code>build.cssMinify = 'esbuild'</code> (so Vite 8's Lightning CSS minifier doesn't strip <code>@layer</code> declarations), and injects <code>&lt;style&gt;@layer theme, base, components, utilities;&lt;/style&gt;</code> into <code>&lt;head&gt;</code> as a runtime failsafe. Setup becomes:

    ```ts
    import { defineConfig } from "vite";
    import plexui from "@plexui/ui/vite";

    export default defineConfig({ plugins: [plexui()] });
    ```

    Your <code>main.css</code> then only needs the Tailwind, Plex UI, and source imports — no manual <code>@layer</code> line. <code>vite</code> is an optional peer dependency (range: 4.x–8.x); if you don't use Vite, you can ignore this export. Layer-order injection can be disabled with the <code>injectLayerOrder: false</code> option. See <a href="/docs/installation">/docs/installation → On Vite</a>.
  </li>
</ul>

## 0.16.0 — 2026-05-02

### Changed

<ul className="changelog-list">
  <li>
    <strong>Default pill shape</strong> on <code>Button</code>, <code>ButtonLink</code>, <code>Select</code>, and <code>SelectControl</code> is now <code>false</code>. The JSDoc on these props always declared <code>@default false</code>; the runtime now matches. Square corners are the modern Plex default and what the Plex docs and Figma kit have shown all along. ## 0.15.3 — 2026-05-02

### Fixed

<ul className="changelog-list">
  <li>
    <strong>@plexui/ui/css distribution</strong> — radius, font, and font-weight tokens are now declared in <code>:root</code> via the new internal <code>variables-tokens-root.css</code>, instead of being pulled in through the <code>@theme static</code> block (which is what 0.15.2 attempted). The <code>@theme static</code> approach worked for component <code>var(--radius-md)</code> lookups but reset Tailwind's own theme via <code>--radius-*: initial</code>, breaking consumer-side utilities like <code>rounded-md</code>, <code>rounded-full</code>, and <code>text-sm</code>. The new file declares the same tokens in <code>:root</code> so components see them but Tailwind's theme stays untouched. The standalone <code>@plexui/ui/styles/variables-tailwind-theme.css</code> sub-export is unchanged and remains opt-in for consumers who explicitly want the Plex scale to drive their Tailwind utilities.
  </li>
</ul>

### Documentation

<ul className="changelog-list">
  <li>
    <strong>Installation guide</strong> (<a href="/docs/installation">/docs/installation</a>) — added an explicit <code>@layer theme, base, components, utilities;</code> declaration to the recommended snippet so cascade layer order is pinned regardless of CSS transformer. New sub-sections cover the Vite + Lightning CSS layer-reorder gotcha (with a one-line workaround to switch the transformer back to PostCSS), an optional <strong>skip-Tailwind-preflight</strong> recipe (Plex UI ships its own reset), and an <strong>opt-in Plex Tailwind theme</strong> recipe for consumers who want their <code>rounded-md</code> utilities to use the Plex radius scale.
  </li>
</ul>

## 0.15.2 — 2026-05-02 — RETIRED

This release is on npm but **should not be used**. Its <code>@theme static</code> import inside <code>@plexui/ui/css</code> resets Tailwind's radius, font, and font-weight defaults to <code>initial</code>, which breaks Tailwind utilities like <code>rounded-md</code>, <code>rounded-full</code>, and <code>text-sm</code> in consumer projects. Upgrade to <strong>0.15.3</strong> immediately.

## 0.15.0 — 2026-04-28

### Added

<ul className="changelog-list">
  <li>
    <strong>TagInput</strong> — new <code>sm</code> size (28&nbsp;px) matching <code>Input</code> and <code>SelectControl</code> sm. Sits flush in horizontal filter rows alongside the matching Select.
  </li>
  <li>
    <strong>TagInput</strong> — new <code>pill</code> prop. Both the container and inner chips become fully-rounded so the control reads as a single rounded surface, matching <code>Input pill</code> and <code>SelectControl pill</code>.
  </li>
  <li>
    <strong>Menu.Content</strong> — exposes <code>onCloseAutoFocus</code>. Call <code>event.preventDefault()</code> to keep the trigger from regaining focus (and its focus-visible ring) after a mouse pick — useful when the menu is paired with a <code>SelectControl</code> in filter rows.
  </li>
</ul>

### Changed

<ul className="changelog-list">
  <li>
    <strong>TagInput</strong> — chip height, font-size, and gutter now scale per-size via <code>--tag-input-tag-&#123;height,gutter,font-size&#125;</code> (defaulting to <code>--tag-input-height</code> / <code>--tag-input-font-size</code>), so a chip-filled container is exactly the same height as an empty one. Previously chip was fixed at <code>--control-size-xs</code> (26&nbsp;px), which made md/lg/xl containers grow when chips were added.
  </li>
  <li>
    <strong>TagInput</strong> — container border-radius now scales per-size (<code>sm=6, md=8, lg=10, xl/2xl/3xl=12</code>), matching <code>SelectControl</code>'s scale. Chip border-radius is concentric with the container (<code>container_radius − inner_padding</code>) at every size.
  </li>
  <li>
    <strong>TagInput</strong> — pill-mode radius is now <code>single_row_height / 2</code> instead of <code>--radius-full</code>. Visually identical when there's one row, but no longer flares into a giant ellipse when chips wrap to a second/third row.
  </li>
  <li>
    <strong>TagInput</strong> — the X close-button inside a chip now has symmetric whitespace on top, right, and bottom (driven by chip's vertical padding), instead of being pulled to the chip edge with <code>margin-right: -1px</code>.
  </li>
</ul>

### Fixed

<ul className="changelog-list">
  <li>
    <strong>TagInput</strong> — added a <code>ResizeObserver</code> on the container and every chip so the internal <code>inputWidth</code> recomputes on any layout change (size/pill toggle, window resize, font load). Without it, a CSS-var-driven reflow didn't trigger React re-renders, which left a stale <code>inputWidth</code> and caused the input or last chip to wrap onto a phantom extra row right after toggling <code>size</code>.
  </li>
</ul>

## 0.14.0 — 2026-04-25

### Added

<ul className="changelog-list">
  <li>
    <strong>Input.AdornmentButton</strong> (<a href="/components/input">Input</a>)<br/>
    New compound sub-component for in-input action buttons (voice toggle, copy, send, visibility). Picks up the same <code>--input-clear-size</code> / <code>--input-clear-pill-offset</code> / <code>--input-clear-icon-size</code> tokens as the built-in clear button, so it stays perfectly aligned inside any <code>Input</code> regardless of size or pill shape. Supports <code>aria-pressed</code> for toggle state and inherits all native <code>&lt;button&gt;</code> props. A new <code>color</code> prop (<code>caution</code> / <code>danger</code> / <code>success</code> / <code>info</code> / <code>discovery</code>) tints the pressed state with Plex's semantic palette — same tokens <code>Button</code> uses — so a recording-mode mic reads as amber, a destructive retry as red, etc.
  </li>
  <li>
    <strong>ButtonGroup component</strong> (<a href="/components/button-group">ButtonGroup</a>)<br/>
    Wraps any row of controls — <code>Button</code>, <code>Input</code>, trigger buttons — and strips the shared border so they read as one unit. Compound API with <code>ButtonGroup.Separator</code> and <code>ButtonGroup.Text</code> (muted label pill; <code>asChild</code> supported). Nest <code>ButtonGroup</code> inside itself for visually separated sub-groups (gap driven by <code>--button-group-nested-gap</code>). Supports <code>orientation="horizontal" | "vertical"</code> for toolbars, steppers, split buttons, pager + arrow nav, rich message composers, and search affordances.
  </li>
  <li>
    <strong>Accordion component</strong> (<a href="/components/accordion">Accordion</a>)<br/>
    Compound primitive built on Radix <code>Accordion.Root</code> — compose <code>Accordion.Item</code>, <code>Accordion.Trigger</code>, <code>Accordion.Content</code>. Supports <code>type="single" | "multiple"</code>, <code>collapsible</code>, per-item <code>disabled</code>, and controlled <code>value</code>/<code>onValueChange</code>. Styled through <code>--accordion-item-divider-color</code>, <code>--accordion-trigger-padding-block</code>, <code>--accordion-icon-size</code>, and a full set of typography / transition tokens. Use it for FAQ sections, settings panels, and collapsible reference lists.
  </li>
  <li>
    <strong>Progress component</strong> (<a href="/components/progress">Progress</a>)<br/>
    Linear progress bar built on Radix <code>Progress.Root</code>. Accepts <code>value</code> (0–<code>max</code>) or <code>null</code> for an indeterminate sweep. Styled through <code>--progress-height</code>, <code>--progress-track-color</code>, <code>--progress-indicator-color</code>, <code>--progress-radius</code>, <code>--progress-indeterminate-duration</code>. Use it for uploads, onboarding completion, and any determinate "X of Y" feedback; <a href="/components/indicators">CircularProgress / LoadingIndicator</a> stay for loading-spinner cases.
  </li>
</ul>

### Changed

<ul className="changelog-list">
  <li>
    <strong>Input</strong> — the <code>disabled</code> state no longer fades the entire Container. The opacity now applies to the input text and the built-in Clear button only, so any <code>Input.AdornmentButton</code> nested via <code>endAdornment</code> stays at full opacity and remains interactive (e.g. a voice-mode toggle that puts the input into a read-only recording state while itself staying clickable).
  </li>
</ul>

<ul className="changelog-list">
  <li>
    <strong>Select options with descriptions auto-bold title</strong> (<a href="/components/select#with-descriptions">Select</a>)<br/>

  <li>

Also replaces the <code>:last-child</code> match (which was also hitting the start-position slot because text nodes don&apos;t count as element children) with an explicit <code>data-position=&quot;end&quot;</code> attribute.

  <li>
    <strong>Avatar demos migrated to shared <code>data-demo-controls</code> pattern</strong> (<a href="/components/avatar">Avatar</a>)<br/>

Overflow count demo grew an <code>overflowCount</code> slider, Colors &amp; variants and Roundness gained <code>size</code> sliders. Colors is now a vertical 2-column grid (<code>soft</code> / <code>solid</code> × 6 colors) so it never clips at narrow widths.
  </li>
</ul>

### Fixed

<ul className="changelog-list">
  <li>
    <strong>Slider hidden in docs <code>SliderValueControl</code></strong><br/>
    A <code>:last-child</code> rule in <code>SliderValueControl</code> was hiding the slider itself when <code>&lt;Slider&gt;</code> rendered without a label (regression from 0.7.35 when Slider&apos;s label row became conditional). The only child of <code>SliderContainer</code> matched both <code>:first-child</code> and <code>:last-child</code>, and the hide rule won. Added <code>:not(:first-child)</code> so the rule only targets the marks container when it exists. All size / roundness / overflow sliders on Avatar, Select, Alert, and Button docs are visible again.
  </li>
</ul>
