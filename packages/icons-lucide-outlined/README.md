# @plexui/icons-lucide-outlined

Lucide icons converted to outlined-fill geometry. Same visual as upstream
[Lucide](https://lucide.dev), but each glyph is a single filled `<path>`
instead of an open stroke.

## Why

Upstream Lucide ships every icon as an open stroke path with
`stroke-width="2"`, `stroke-linecap="round"`, `stroke-linejoin="round"`.
That works great for most cases, but it has trade-offs:

- The visual thickness is tied to a `stroke-width` attribute. Scaling the
  icon either keeps strokes pixel-aligned (`vector-effect: non-scaling-stroke`)
  or scales them with the shape — both have caveats.
- Tinting requires setting `stroke` (not `fill`); design systems that
  standardize on fill-based tinting need a separate code path.
- Animations that morph fill regions can't target the glyph silhouette
  directly.

The outlined variant fixes all three by **converting each stroke into the
filled outline of that stroke** — geometrically identical at the design
size, but rendered as a single closed `<path d="…" fill="currentColor" />`.

## Source & generator

Icons are generated by `scripts/build-lucide-outlined.mjs` at the repo
root, which runs `inkscape --actions="select-all;object-stroke-to-path"`
across every SVG in `node_modules/lucide-static/icons/`. The output of
each run lands in `icons/<slug>.svg`. License inherits from upstream
Lucide (ISC).

## Usage in the docs site

The catalog generator (`scripts/build-icon-catalogs.mjs`) reads from
`packages/icons-lucide-outlined/icons/` and emits
`public/icon-libraries/lucide-outlined.json`, which the IconBrowser
fetches at runtime. See `/icons/lucide` → "Outlined Stroke" tab.

## Regenerating

```bash
brew install --cask inkscape   # one-time, ~700 MB
npm --workspace=@plexui/icons-lucide-outlined run build
```

The build is idempotent — re-running it just overwrites `icons/`.
