# @plexui/icons-hugeicons-outlined

Hugeicons free icons converted to outlined-fill geometry. Same visual as
upstream [Hugeicons](https://hugeicons.com), but each glyph is a single
filled `<path>` instead of an open stroke.

## Why

Upstream Hugeicons ship as element-descriptor arrays for the
`<HugeiconsIcon>` React component, rendered as open stroke paths. That
matches the Lucide / Tabler conventions and works great for most cases,
but it has trade-offs:

- The visual thickness is tied to a `stroke-width` attribute. Scaling
  the icon either keeps strokes pixel-aligned
  (`vector-effect: non-scaling-stroke`) or scales them with the shape —
  both have caveats.
- Tinting requires setting `stroke` (not `fill`); design systems that
  standardize on fill-based tinting need a separate code path.
- Animations that morph fill regions can't target the glyph silhouette
  directly.

The outlined variant fixes all three by **converting each stroke into the
filled outline of that stroke** — geometrically identical at the design
size, but rendered as a single closed `<path d="…" fill="currentColor" />`.

Mixed-mode Hugeicons (descriptors that bake fill geometry into stroke
icons) are skipped — the same set the catalog generator already
filters. See `scripts/build-icon-catalogs.mjs#isMixedHugeicon`.

## Source & generator

Icons are generated by `scripts/build-hugeicons-outlined.mjs` at the
repo root. For each pure-stroke icon descriptor exported from
`@hugeicons/core-free-icons`, the script:

1. Renders the descriptor to a temporary SVG (matching the catalog
   builder's `hugeiconArrayToSvg` shape — 24×24, `stroke-width="2"`,
   round caps + joins).
2. Runs `inkscape --actions="select-all;object-stroke-to-path"` to
   flatten the stroke into a filled path.
3. Normalises the result to a single `<path d="…" fill="currentColor"/>`
   inside a 24×24 viewBox SVG.
4. Writes to `icons/<exportName>.svg` (e.g. `SearchIcon.svg`).

## Usage in the docs site

The catalog generator (`scripts/build-icon-catalogs.mjs`) reads from
`packages/icons-hugeicons-outlined/icons/` and emits
`public/icon-libraries/hugeicons-outlined.json`, which the IconBrowser
fetches at runtime. See `/icons/hugeicons` → outlined toggle in the
detail panel.

## Regenerating

```bash
brew install --cask inkscape   # one-time, ~700 MB
npm --workspace=@plexui/icons-hugeicons-outlined run build
```

The build is idempotent — re-running it just overwrites `icons/`.
